# 06_跨集一致性审查 — 每 5 集一次的防漂移机制

> 不审查,你的第 20 集主角的脸跟第 5 集会差一个人。
> 这不是危言耸听,是 80 集 AI 长剧的物理规律。
> 这份文档讲怎么把 90 分钟的审查跑出来,防止漂移。

================================================================
# 为什么必须每 5 集审一次
================================================================

## 漂移是数学问题,不是态度问题

每集 AI 生成都会有"微小漂移"(肉眼可能看不出来,但确实存在)。
假设每集漂移 1%,那么:

- 第 5 集 vs 第 1 集:累积 5% 漂移(肉眼难辨)
- 第 10 集 vs 第 1 集:累积 10% 漂移(开始可见)
- 第 20 集 vs 第 1 集:**累积 20% 漂移(明显变脸)**
- 第 50 集 vs 第 1 集:**累积 50% 漂移(完全不像)**

**单向漂移不可逆。** 一旦漂移就是漂移了,只能"重新校准"回基线。

## 审查的本质是"把基线拉回来"

每 5 集审查一次,把漂移控制在 5% 以内,80 集累积不会超过 5%。
**这是唯一能跑通 80 集的方法。**

## 不审查会怎样(真实场景)

- 第 30 集你发现主角的鼻子有点不一样了,你想"是 AI 偶尔抽风吧"
- 第 35 集你发现主角的眉间朱砂痣位置偏了 0.2cm,你想"差不多吧"
- 第 40 集你回看第 1 集,发现根本不是同一个人 → **崩溃**
- 这时候你已经投了 200+ 小时,只能选择:
  - 接受漂移,降低质量标准(对不起观众)
  - 回去重做(浪费 200 小时)
  - 放弃项目(常见结局)

**不审查 = 50% 概率项目崩盘。审查 = 防崩盘保险。**

================================================================
# 审查的时机和范围
================================================================

## 强制审查时机

| 时机 | 审查范围 | 工时 |
|---|---|---|
| 第 5 集完成后 | EP01-05 | 1.5 h |
| 第 10 集完成后 | EP06-10 + 与 EP01-05 对比 | 1.5 h |
| 第 15 集完成后 | EP11-15 + 与 EP01-10 对比 | 1.5 h |
| 第 20 集完成后(篇章交界) | EP01-20 全审 | 3.0 h |
| 第 25 集后 | EP21-25 + 与 EP20 对比 | 1.5 h |
| ... | 每 5 集一次 | ... |
| 每篇章结束 | 整篇章全审 | 3.0 h |

## 关键节点强化审查
- 篇章交界(第 20、40、60 集):全篇章审查
- 重大事件后(主角进阶 / 服装切换):专项审查
- 异常迹象出现时(肉眼看到漂移):立即审查

================================================================
# 标准审查流程(90 分钟,5 集量)
================================================================

## Step 1 — 截图采集(15 分钟)

为本批次每集采集 **6 张关键截图**:

```
对每个主角(CHAR-A, CHAR-B):
  - 1 张面部特写(中近景以上)
  - 1 张全身镜头
  - 1 张服装细节

对每个核心场景(LOC-01, LOC-02):
  - 1 张全景

对每个核心法器(PROP-SWORD-01):
  - 1 张特写
```

5 集 × 平均 6 张 = 30 张,放入临时审查目录:
```
10_跨集审查报告/审查_EP06-10_2026-XX-XX/
  原图/
    EP06_CHAR-A_face_U03.png
    EP06_CHAR-A_body_U01.png
    EP06_LOC-01_wide_U02.png
    EP06_PROP-SWORD-01_close_U08.png
    EP07_...
    ...
```

## Step 2 — 基线对比(30 分钟)

把这 30 张图与 **基线参考图** 并排对比:

```
基线 = 角色基因库的定妆图 + 资产模块库的标准参考图
```

对每张图问 4 个问题:
1. **是否符合 JSON 中的视觉锚点?**
2. **颜色是否在 ±15% 漂移范围内?**
3. **比例 / 形态是否一致?**
4. **是否触发任何"绝对禁忌"?**

工具:用图片对比软件(或简单的"PPT 并排放")。

记录到审查表:

```markdown
# 审查报告 — EP06-10 / 2026-XX-XX

## 角色一致性
| 集 | 角色 | 锚点1朱砂痣 | 锚点2耳坠 | 锚点3戒指 | 整体相似度 |
|---|---|---|---|---|---|
| EP06 | CHAR-A | ✅ 准 | ✅ 准 | ✅ 准 | 95% |
| EP07 | CHAR-A | ⚠️ 偏左 1mm | ✅ 准 | ✅ 准 | 90% |
| EP08 | CHAR-A | ✅ 准 | ❌ 双耳都有 | ✅ 准 | 80% |
| EP09 | CHAR-A | ✅ 准 | ✅ 准 | ✅ 准 | 92% |
| EP10 | CHAR-A | ⚠️ 偏小 | ✅ 准 | ⚠️ 不清晰 | 85% |

## 资产一致性
| 集 | 资产 | 颜色 | 形状 | 装饰 | 总评 |
|---|---|---|---|---|---|
| EP06 | OUTFIT-A1 | ✅ | ✅ | ✅ | 95% |
| EP07 | OUTFIT-A1 | ⚠️ 偏蓝 | ✅ | ✅ | 88% |
| ... | | | | | |
```

## Step 3 — 趋势分析(15 分钟)

把本批次的相似度数据 + 历史所有批次的数据,画一条趋势线:

```
CHAR-A 相似度趋势(基线=100%):
EP01: 100% (基线)
EP05: 96%
EP10: 92%
EP15: 88%   ← 在下降
EP20: ?     ← 警报区:必须重新校准
```

判断规则:
- **绿区**(≥ 90%):正常,继续
- **黄区**(85-90%):警告,本批次有问题但还可控
- **红区**(< 85%):必须停工校准,不允许做下一集

## Step 4 — 漂移定位(20 分钟)

对所有黄/红区的项目,定位漂移的源头:

```
漂移类型 1:角色基因 JSON 没遵守
→ 检查生成提示词,是否每次都贴了校验提示词
→ 检查参考图,是否用了过期版本

漂移类型 2:多次重抽导致带坏
→ 检查 11_失败池/,看 ABC 三版差异
→ 锁定"够好就定"的纪律

漂移类型 3:同框混脸
→ 检查关系矩阵 JSON 是否落实

漂移类型 4:篇章切换没更新
→ 检查是否到了换服装时间(第 16 集 OUTFIT-A1 → A2),却还在用旧参考

漂移类型 5:AI 工具本身偏移(很少见)
→ 几乎不可能,但发生时换工具版本试一下
```

## Step 5 — 行动决策(10 分钟)

根据漂移定位,做 3 类行动:

### 行动 A:补丁修复(轻度漂移)
- 在 `项目状态.json` 加 TODO,下次出图时强制注意
- 不需要重做已完成集

### 行动 B:基因更新(中度漂移)
- 更新角色基因 JSON,推到新版本(如 v1.2)
- 更新校验提示词
- 标记从下一集开始用新版本

### 行动 C:重做(重度漂移)
- 重做漂移最严重的 1-2 集的关键镜头
- 这是最后手段,工时成本高

# 篇章交界的强化审查(每 20 集)
================================================================

## 普通审查 vs 篇章审查的差异

| 维度 | 5 集审查 | 篇章审查 |
|---|---|---|
| 时间 | 90 分钟 | 3 小时 |
| 范围 | 最近 5 集 | 整篇章 |
| 深度 | 视觉一致性 | 视觉 + 剧情 + 资产生命周期 |

## 篇章审查的 6 个额外维度

### 1. 角色弧光验收
对照圣经第⑪角色弧光分集表:
- 主角的情绪 / 状态变化是否按预定节奏走?
- 配角的进入 / 退出时间是否准确?

### 2. 资产生命周期验收
对照圣经第⑫资产引入计划 + `资产引用追踪.json`:
- 该退役的资产退役了吗?
- 该启用的新资产启用了吗?
- 没用的资产是否累积过多?

### 3. 色调子主题验收
对照圣经第⑤美术方向的"不同篇章的色调子主题":
- 本篇章整体色调是否符合规划?
- 与上一篇章的色调过渡是否自然?

### 4. 情绪弧线验收
对照圣经第⑦情感弧线表:
- 整篇章情绪是否在预定区间?
- 关键节点(第 5/10/15/20)情绪是否到位?

### 5. 工时和效率验收
对照状态机的工时记录:
- 本篇章平均单集工时多少?
- 哪些集异常超时?为什么?
- 流程优化点?

### 6. TODO 清理验收
- 本篇章累积的 TODO 处理了多少?
- 哪些 TODO 推到下一篇章?

## 篇章审查后的"圣经更新"

篇章交界是**唯一允许更新圣经主体的时机**。

```
篇章 1 结束:
1. 完成篇章审查报告
2. 整理本篇章累积的"已锁定决策"
3. 更新圣经到 v2.0:
   - 第⑨剧集总弧:记录第一里程碑达成
   - 第⑩篇章:篇章 1 结束,篇章 2 启动设置
   - 第⑪角色弧光:修正不准确的预设
   - 第⑫资产引入计划:篇章 2 资产引入排程
4. 备份当前状态到 状态快照/
5. 启动篇章 2
```

# 自动化辅助:一致性检查脚本
================================================================

`脚本/一致性检查.py` 可以做的事:

```python
# 检查项 1:ID 引用完整性
# - 圣经里提到的 ID 是否都在基因库 / 资产库中存在?

# - 哪些资产 N 集没用了?

# 检查项 4:红线警报
# - 单集工时 > 12h
# - 距上次审查 ≥ 5 集
# - 高优 TODO ≥ 5
```

跑法:
```bash
cd 项目根目录
python3 ../长剧玄幻流水线/脚本/一致性检查.py
```

输出报告 + 自动触发警报。

# 审查者心态(单人审自己很难)
================================================================

## 难点
- 你既是创作者又是审查员,容易"自我宽恕"
- 看自己的作品越看越觉得"还行"

## 应对

### 技巧 1:延迟审查
做完第 10 集 1 天后再审,不要立刻审。
**昨天的你已经不是今天的你了**,有距离才有审查的客观性。

### 技巧 2:量化优先
不用"看起来差不多",用相似度百分比。**数字不会撒谎。**

### 技巧 3:对照基线,不对照前一集
- 错:第 10 集 vs 第 9 集 → 看起来一样,没问题
- 对:第 10 集 vs 第 1 集 → 累积 10% 漂移,要注意

### 技巧 4:听一个外部人意见
每 20 集让朋友 / 家人看一遍,问:
"主角第 1 集和第 20 集是同一个人吗?"
**如果他们犹豫,就是问题。**

### 技巧 5:看跨度大的截图组
打开 5 张截图:EP01 / EP05 / EP10 / EP15 / EP20
**并排看,漂移立刻显形。**

# 审查的"最小成本版本"
================================================================

如果实在没时间做完整审查,做最小版本(30 分钟):

```
[ ] 打开主角 EP01 / EP当前 的面部特写,并排对比
[ ] 检查朱砂痣 / 耳坠 / 颈纹 3 个最关键的锚点
[ ] 检查服装颜色是否同色
[ ] 检查法器是否一致
[ ] 如果以上都过 → 绿灯
[ ] 任何一项有问题 → 黄灯,登记 TODO
[ ] 任何一项严重 → 红灯,做完整审查
```

**最小审查比不审查强 100 倍。** 每 5 集做一次最小版本,工时只增加 30 分钟。

# 常见坑
================================================================

### 坑 1:跳过审查
"这批集看起来还可以,跳过这次审查吧" → 漂移开始累积。
**审查是纪律,不是感觉。该审就审。**

### 坑 2:只看好的样本
专挑成功的截图审查 → 假绿灯。
**必须随机采样,包括明显有问题的镜头。**

### 坑 3:发现问题不行动
审查报告写了"主角朱砂痣偏移",但不更新基因 / 不加 TODO → 等于没审。
**审查必须产出行动,不产出行动的审查是浪费时间。**

### 坑 4:绿灯标准太松
"差不多就算绿灯吧" → 标准滑坡,最终崩盘。
**90% 相似度是绿灯,89% 就是黄灯,严格执行。**

### 坑 5:不留快照
"我审查了,我记得"→ 半年后回看不知道审查过什么。
**所有审查报告必须存盘,审查报告也是项目资产。**
