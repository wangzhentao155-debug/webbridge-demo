# 05_单集 8 小时压缩流程 — 单人扛 80 集的核心工序

> 这一份是你每天都要用的。
> 80 集每集 8-12 小时,超 12 小时 = 流程出问题,不是你不够努力。
> 这份流程是把 25 小时(单片节奏)压到 8 小时的工程方法。

================================================================
# 为什么必须压到 8 小时
================================================================

## 工时算账

| 节奏 | 单集时长 | 80 集总工时 | 全职(8h/天) | 业余(3h/天) |
|---|---|---|---|---|
| 不压缩(《猎犬》节奏) | 25 h | 2000 h | 250 天 ≈ 1 年 | 667 天 ≈ 2 年 |
| 压到 12 小时 | 12 h | 960 h | 120 天 ≈ 4 月 | 320 天 ≈ 11 月 |
| **压到 8 小时(目标)** | **8 h** | **640 h** | **80 天 ≈ 3 月** | **213 天 ≈ 7 月** |

**8 小时 vs 25 小时,差出来的 17 小时去哪了?**
- 资产复用(不再每集重做角色)→ 省 5h
- 模板填空(不再每集重写提示词)→ 省 4h
- 基因校验自动化(不再每张图人工对锚点)→ 省 3h
- 批量并行生成(不再串行等)→ 省 3h
- 不重做超过 3 次(纪律)→ 省 2h

**任何一项偷懒,工时立刻反弹。**

# 阶段 1:单集大纲(0.5 h)
================================================================

## 目标
把当集的剧情写成可拍摄的镜头单元清单。**不写小说,写剧本拍摄计划。**

## 输入
- `01_圣经/圣经_v当前版本.md` 第⑩篇章 + 第⑪角色弧光
- 上一集结尾的"承接出"
- `项目状态.json` 中的已锁定决策

## 产出
`04_单集大纲/EP[NN]/大纲.md`,4 个字段填完:

```markdown
# EP[NN] — [集名]

## 1. 一句话本集功能
本集让观众 [理解 / 感受 / 怀疑 / 期待] 什么?
30 字以内,说不清就重写。

## 2. 单元清单(8-15 个单元,每单元 10-15 秒)
| 单元 | 时长 | 功能 | 涉及角色 | 涉及场景 | 涉及资产 |
|---|---|---|---|---|---|
| U01 | 12s | 承接上集,主角入场 | CHAR-A | LOC-01 | PROP-01 |
| U02 | 15s | 反派出现 | CHAR-A, CHAR-B | LOC-01 | - |
| ... | | | | | |

## 3. 承接入(本集开头)
延续 EP[NN-1] 结尾的 [什么动作/视线/物件]。

## 4. 承接出(本集结尾)
本集结尾留下 [什么动作/视线/物件] 给 EP[NN+1] 接。
```

## 红线
- 单集单元数 ≤ 15(超过 → 本集剧情太密,拆成两集)
- 涉及新场景 ≤ 1(超过 → 资产爆炸,合并到已有场景)
- 涉及新角色 ≤ 1(超过 → 看圣经⑪角色弧光,本集是否真该来这么多)

## 时间预算
- 思考 + 写:30 分钟
- 超时 → 说明圣经第⑩篇章没规划好,回去补

================================================================
# 阶段 2:分镜规划(1.0 h)
================================================================

## 目标
为每个单元写"导演分镜蓝图"——景别 + 运镜 + 灯光 + 情绪。

## 输入
- 阶段 1 的大纲
- `02_角色基因库/校验提示词.md`(本集涉及角色的)
- `03_资产模块库/LOC-XX.json`(本集场景的)

## 产出
`04_单集大纲/EP[NN]/分镜规划.md`,每个单元 4 格:

```markdown
## U01 — [单元标题](00:00-00:12)
场景:LOC-01 ｜ 角色:CHAR-A ｜ 资产:PROP-01

### c1 [镜头标题](3s)
景别:中景
导演构思:[一句话]
细节:[关键道具/角色锚点在画面里如何呈现]
CAMERA:[焦段,机位高度]
MOVEMENT:[Static / Slow Push / Tracking / Handheld]
AUDIO:[环境声 / 配乐 / 音效]
MOOD:[情绪关键词]

### c2 [镜头标题](3s)
...
### c3 [镜头标题](3s)
...
### c4 [镜头标题](3s)
...

转场钩子:[本单元结束如何接 U02]
```

## 关键纪律(吸收《猎犬》方法论)
- 每个单元必有一个"记忆点"(让观众记得住的画面)
- 景别跟着情绪走(远景=孤独,特写=内心)
- 同框 ≤ 3 人(古装玄幻铁律)
- 每单元的灯光基调必须和篇章色调一致

## 时间预算
- 每单元 5-8 分钟,15 个单元 ≈ 1 小时
- 超时 → 套模板填空,不要重新发明轮子

================================================================
# 阶段 3:分镜板出图(1.5 h)
================================================================

## 目标
每个单元出一张四格分镜板。这是视频生成的视觉锚点。

## 输入
- 阶段 2 的分镜规划
- 角色基因库的参考图(定妆图 + 三视图 + 表情)
- 资产模块库的参考图(服装 + 场景 + 法器)
- `02_角色基因库/CHAR-X/校验提示词.md`

## 产出
`05_分镜板/EP[NN]/U[NN]_分镜板.png`,每个单元一张。

## 提示词组装公式
```
[校验提示词 - 从基因库复制] 
+ 
[场景模块的强制约束 - 从 LOC-XX.json 复制] 
+ 
[阶段 2 写的 4 格内容]
+
[底部统一约束:16:9 / 网格化 / 电影分镜板风格]
```

## 标准模板见 `模板/分镜提示词.md`

## 时间预算
- 每张分镜板:写提示词 2 分钟 + 上传参考图 1 分钟 + 生成等待 2-3 分钟 + 验收 2 分钟
- 单张约 7-10 分钟,15 张 ≈ 1.5 小时

## 加速技巧
- **批量打开 5 个浏览器标签页并行生成**
- 验收按"基因 JSON 的禁忌清单"逐条对照,不靠肉眼凭感觉
- 重做硬上限 3 次,第 4 次重写镜头不死磕

================================================================
# 阶段 4:视频提示词(1.0 h)
================================================================

## 目标
为每个单元写 Seedance 生成提示词。

## 关键纪律
**视频提示词只写"这 15 秒怎么活",不重复"画面里有什么"。**
画面是分镜板的事,视频是导演运镜的事。

## 标准模板见 `模板/seedance批次.md`

## 提示词组装公式
```
[共享部分 - 整集复用]
  - 一致性铁律(从圣经第⑧节复制)
  - 角色一致性锁定(从基因库复制)
  - 必须上传的参考图清单
  
+

[逐镜部分 - 每镜不同]
  - 时间线 0-Xs:每秒做什么
  - 运镜方式:Static / Slow Push / Tracking 等
  - 灯光指示(从分镜板复制)
  - 情绪关键词
  - 开场衔接:从上镜什么动作接来
  - 收尾衔接:本镜结束什么给下镜接
```

## 时间预算
- 共享部分一次写完:15 分钟
- 每个单元逐镜部分 3-5 分钟,15 个 ≈ 45 分钟
- 合计约 1 小时

================================================================
# 阶段 5:视频生成(2.0 h)
================================================================

## 目标
跑 Seedance,生成 15 个单元的视频(约 15 段 × 12-15 秒)。

## 执行顺序(吸收 ChatGPT SOP 的批量顺序)
```
1. 先做低风险情感/静态单元(拿信心+样本)
   - 单人镜头
   - 静止动作
   - 室内中景
   
2. 再做动作单元
   - 武打 / 法术释放
   - 御剑飞行
   
3. 最后做最难的多角色/复杂运动单元
   - 同框 2-3 人
   - 法术对决
   - 异兽出场
```

**为什么这个顺序:** 
- 低风险单元跑通了,Seedance 在你这个项目上的稳定参数就找到了
- 用这些参数再去做高风险单元,翻车率显著降低
- 心理上也容易:先看到 8 个成功的视频,再啃 2 个难的

## 并行策略
**单镜串行做要 5 分钟 × 15 = 75 分钟。**
**5 个标签页并行做 = 15 分钟搞定 5 镜,75 → 25 分钟。**

```
浏览器标签 1: U01, U06, U11
浏览器标签 2: U02, U07, U12
浏览器标签 3: U03, U08, U13
浏览器标签 4: U04, U09, U14
浏览器标签 5: U05, U10, U15
```

每个标签页跑 3 个,5 标签并行,总用时 ≈ 15 分钟生成 + 验收时间。

## 验收清单(每段视频)
```
[ ] 主角脸跟基因库基本一致(允许 ±10%)
[ ] 视觉锚点至少 1-2 个清晰可见
[ ] 没有明显 AI 鬼脸 / 鬼手
[ ] 一镜到底,无内部跳剪
[ ] 没有字幕 / 旁白 / 屏幕乱码
[ ] 颜色没有超出篇章主调
[ ] 运镜符合分镜规划
```

## 返修三原则(吸收 ChatGPT SOP 的精华)
```
能剪就别重做
能裁就别重生  
能用音效遮就别重抽
```

**单镜重做硬上限 3 次。** 第 4 次重写镜头,不死磕。

## 时间预算
- 并行生成时间 ≈ 30-45 分钟
- 逐段验收 ≈ 30-45 分钟
- 返修(预估 1-2 段需要返修) ≈ 30 分钟
- 合计约 2 小时

================================================================
# 阶段 6:剪辑(1.5 h)
================================================================

## 目标
把 15 段视频剪成一集成片。

## 工序(剪映 / 达芬奇)

### Step 1 — 导入排序(10 分钟)
- 新建项目,1080P,24fps
- 按 U01 → U15 顺序拖入
- 首尾各切 0.5-1 秒(Seedance 可能"展示参考图")

### Step 2 — 衔接处理(20 分钟)
- 默认硬接
- 跳切生硬处 0.3 秒叠化(不超过 2 处)
- 检查"承接入"和"承接出"的视觉扣环是否对上

### Step 3 — 字幕(20 分钟)
- 关键剧情台词加字幕(白色 / 居中 / 底部)
- 简洁无衬线,无花字
- 数量克制

### Step 4 — 音乐音效(15 分钟)
- 大部分保留 Seedance 原声
- 情绪点淡入极简音乐(ambient / piano)
- 严禁强鼓点抢戏

### Step 5 — 调色统一(15 分钟)
- 全集套统一基调(对照篇章色调)
- 检查暖色 / 强调色是否只在指定情境出现
- 检查跨单元颜色无突变

### Step 6 — 片头片尾(10 分钟)
- 第 1 集:完整片头(3 秒标题)
- 其他集:logo 一闪(1 秒)即可,不要重复完整片头
- 片尾:留白 + 下集预告(可选)

### Step 7 — 导出(10 分钟)
- MP4,1080P+,24fps
- 命名:`EP[NN]_[集名]_v01.mp4`
- 放入 `09_成片导出/`

## 时间预算
- 合计约 1.5 小时
- 第一集会更慢(2-3 小时),第 5 集起稳定到 1.5 小时

================================================================
# 阶段 7:收尾 / 状态更新(0.5 h)
================================================================

## 目标
- 完整看一遍成片
- 更新状态机
- 登记发现的 TODO

## 操作清单(15 分钟)
```
[ ] 完整看一遍成片(15 分钟)
    - 第一遍看流畅度
    - 不要细抠,看整体是否能"过"
[ ] 更新 项目状态.json(10 分钟)
    - 各集状态:进行中 → 已完成
    - 进度概览:已完成集数 + 1
    - 工时记录:本月_已用小时 += 8
    - 资产清单:新增/退役的资产
    - TODO列表:发现的问题登记
[ ] 备份到云端(自动化,5 分钟)
```

## 红线检查
完成本集后跑一次 `脚本/一致性检查.py`,如果触发红线:
- 单集工时 > 12h
- 距上次跨集审查 ≥ 5 集
- TODO 高优先级 > 5 个

→ **不开下一集,先处理红线。**

================================================================
# 哪些阶段最容易爆时(预防)
================================================================

## 容易爆时排行榜

### 第 1 名:阶段 3 — 分镜板出图(爆时 70%)
**爆时原因:** 重做超过 3 次还不甘心
**应对:** 严格 3 次上限,接受"够好"

### 第 2 名:阶段 5 — 视频生成(爆时 50%)
**爆时原因:** 不并行 / 不按顺序 / 不接受小瑕疵
**应对:** 5 标签并行 + 返修三原则

### 第 3 名:阶段 6 — 剪辑(爆时 30%)
**爆时原因:** 想做"电影感转场"
**应对:** 硬接为主,告诉自己"剪辑师不是导演"

### 不太爆的:阶段 1/2/4/7
**这些是文字工作,模板填空,有上限。**

## 一句话防爆:
**所有阶段都有"够好就过"的指标,不要追求单一阶段完美。**

# 第一集慢是正常的

- 第 2 集:10-12 小时
- 第 3 集:9-10 小时


================================================================
# 单集流程的"红黄绿灯"
================================================================

## 🟢 绿灯(继续下一集)
- 本集工时 8-10 小时
- 验收无重大问题
- 状态机更新完毕
- 跨集审查间隔在 5 集内

## 🟡 黄灯(暂停一天,复盘)
- 本集工时 10-12 小时
- 有 2+ 个 TODO 未处理
- 资产开始漂移迹象
- 心态出现疲劳

## 🔴 红灯(必须停工解决)
- 本集工时 > 12 小时
- 跨集审查发现严重一致性问题
- 高优先级 TODO ≥ 5 个
- 连续 3 集都超时

**红灯时硬上下一集 = 雪球越滚越大,80 集做到一半崩溃。**
